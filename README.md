# Мафия — ИИ-ведущий

Мини-приложение (веб, адаптив под мобилку / Telegram Web App) для игры в мафию без живого ведущего: **ИИ полностью ведёт игру** и говорит от лица ведущего (озвучка с телефона создателя комнаты на колонку).

## Идея

- На празднике собирается компания, хочется поиграть в мафию, но быть ведущим никому не хочется.
- Один человек открывает приложение, **создаёт комнату** и получает **код приглашения** (например, 6 цифр).
- Остальные вводят код и **подключаются** к комнате.
- Создатель комнаты нажимает «Начать игру» и **подключает телефон к колонке** — дальше говорит только ИИ-ведущий.
- ИИ распределяет роли, объявляет ночь/день, даёт слово игрокам, проводит голосование и объявляет результат — **полная замена ведущего**.

## Роли (базовый набор)

| Роль       | Количество | Действие ночью        |
|-----------|------------|------------------------|
| Мафия     | 1–2        | Выбирает жертву        |
| Доктор    | 1          | Выбирает, кого спасти  |
| Детектив  | 1 (опц.)   | Проверяет одного на мафию |
| Мирные    | остальные  | —                      |

Победа: мафия выигрывает, когда мафий ≥ мирных; мирные — когда все мафии выбыли.

## Фазы игры

1. **Лобби** — ожидание игроков, создатель запускает игру.
2. **Раздача ролей** — каждому в личку (на экране) показывается его роль.
3. **Ночь** — по очереди: мафия выбирает жертву, доктор — кого спасти, детектив — кого проверить. ИИ объявляет «Закройте глаза», «Мафия, откройте глаза» и т.д.
4. **День** — ИИ объявляет, кто погиб (или что ночь прошла спокойно), даёт время на обсуждение, затем голосование.
5. **Голосование** — игроки голосуют за исключение; ИИ объявляет результат и исключённого.
6. Повтор 3–5 до победы мафии или мирных.

## ИИ-ведущий

- Все реплики ведущего генерируются на бэкенде (шаблоны фраз под фазу/событие).
- На клиенте они показываются **текстом** и при необходимости **озвучиваются** (Web Speech API или внешний TTS), чтобы создатель комнаты мог вывести звук на колонку.
- Формально «говорит» ИИ от имени ведущего — один голос на всю игру.

## Технологии (тестовая реализация)

- **Backend:** Node.js, Express, Socket.io (комнаты, коды, состояние игры, реплики ведущего).
- **Frontend:** адаптивная вёрстка, экран создания/входа по коду, экран игры с репликами ИИ и кнопками фаз (голосование, выбор жертвы и т.д.).
- **ИИ:** пока правило-ориентированные фразы (русский язык); при желании позже можно подключить LLM для более живых комментариев.

## Запуск

**Режим разработки** (клиент на :5173, бэкенд на :3000, прокси). Нужны **оба** процесса, иначе будет ошибка `ECONNREFUSED 127.0.0.1:3000` (прокси не находит сервер).

**Вариант 1 — одной командой из корня:**
```bash
cd mafia_project
npm install
npm run dev
```
Поднимутся и сервер (порт 3000), и клиент (порт 5173). Открой **http://localhost:5173**.

**Вариант 2 — два терминала:**
```bash
# Терминал 1 — сервер (сначала!)
cd server && npm install && npm run dev

# Терминал 2 — клиент (после появления "Mafia server http://localhost:3000")
cd client && npm install && npm run dev
```

Открой в браузере **http://localhost:5173**. Создай комнату → открой вторую вкладку/телефон и войди по коду → начни игру.

**Озвучка ведущего:** создатель комнаты включает «Озвучивать ведущего» — реплики ведущего озвучиваются (подключи телефон к колонке). Варианты: **OpenAI TTS** (`OPENAI_API_KEY` в `server/.env`) или **ElevenLabs** (лучше для русского; ключ в `server/.env` или через Cloudflare Worker — см. `worker/README.md`). На хостингах вроде Beget ElevenLabs с сервера часто даёт 403; тогда задайте `TTS_CF_WORKER_URL` на развёрнутый Worker (ключ хранится в секретах Worker). Подробнее: `server/.env.example`, `DEPLOY_SERVER.md`.

**Продакшен:** собери клиент (`cd client && npm run build`) и запускай только сервер — он отдаёт статику из `client/dist` и сокеты на порту 3000.

## Тесты (сервер)

В папке `server/`:

- **Быстрые тесты (логика + ведущий + синхронность по коду):**
  ```bash
  cd server && npm install && npm run test
  ```
  Проверяются: распределение ролей, победа, живые/мёртвые, fallback-реплики ведущего без API, **спецификация синхронности** — что в коде порядок вызовов соблюдён: реплика ведущего → пауза → событие игрокам (ночь: `night_turn` после «проснитесь», день: `phase day` после итога ночи и обсуждения, голосование и финал — после соответствующих реплик).

- **Задержки для тестов/настройки:** все паузы читаются из env (например `MAFIA_DELAY_AFTER_HOST_WAKE_MS`, `MAFIA_DELAY_AFTER_NIGHT_SUMMARY_MS`). Для ускорения прогонов можно задать меньшие значения.

- **Интеграционный тест синхронности** (поднимает сервер, подключает клиентов, проверяет порядок событий в реальном времени):
  ```bash
  cd server && RUN_SYNC_INTEGRATION=1 npm run test:sync
  ```
  Требует установленный `socket.io-client` (устанавливается с `npm install` в `server/`).
